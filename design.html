Key Decisions

dart:ffi - Zig compile to C ABI shared library, Flutter calls via FFI. Zero overhead.
Texture Widget - Render NES frame buffer directly to Flutter Texture for 60fps.
Isolate - Emulation loop runs on a separate Dart Isolate to avoid UI jank.
Adaptive UI - Single codebase, responsive for phone & desktop.

Tech Stack & Features
Modern toolkit for a retro experience
💫
Flutter 3.x
Cross-platform UI framework. Material 3 design system with adaptive layouts for phone and desktop.
⚡
Zig Core via FFI
Compile Zig emulator to C ABI .so/.dylib. dart:ffi calls directly into CPU/PPU/APU with zero overhead.
🎨
Texture Rendering
Flutter Texture widget renders NES frame buffer at 60fps. Shared memory buffer between Zig and Dart.
🎶
Low-Latency Audio
Platform audio via method channel. Android AudioTrack / macOS CoreAudio for minimal audio latency.
💾
Save States
Snapshot full emulator state (CPU, PPU, RAM). Multiple save slots with thumbnail previews.
🛠
Debug Tools
CPU register viewer, PPU palette & pattern tables, OAM sprite inspector. Ported from Raylib debug views.

Implementation Roadmap
Phase-by-phase delivery plan
  • 1
    Phase 1: Project Setup & FFI Bridge
    Flutter project scaffold, Zig C-ABI export, dart:ffi bindings, frame buffer shared memory
  • 2
    Phase 2: Core Rendering & Input
    Texture widget 60fps rendering, keyboard input (macOS), virtual gamepad (Android), audio bridge
  • 3
    Phase 3: Library & ROM Management
    ROM file picker, iNES header parsing, game library with thumbnails, search & filter
  • 4
    Phase 4: Save States & Settings
    Save/load state system, persistent settings, CRT/pixel shader filters, favorites
  • 5
    Phase 5: Debug Tools & Polish
    CPU/PPU debug panel, pattern table viewer, OAM inspector, performance optimization
